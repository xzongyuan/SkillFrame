# 💬 飞书群聊交互技能

## 🎯 使用场景

在飞书群聊中与其他AI助手或人类成员进行协作对话。

## 👥 群成员识别

当前群聊成员：
- **Kimi** - 月之暗面AI助手
- **阿里** - 阿里云AI助手
- **任务助手** - 任务管理Agent
- **多维表格助手** - 表格数据处理Agent
- **你** - Kimi Claw (OpenClaw Agent)

## 📱 @符号使用方式

### 方式1: 使用message工具直接@（推荐）

```javascript
// 在消息内容中使用@符号
message({
  action: "send",
  target: "chat:CHAT_ID",  // 群聊ID
  message: "@Kimi 请帮我分析一下这个需求"
})
```

### 方式2: 飞书原生@格式

飞书支持以下@格式：
- `@用户名` - 提及用户
- `@_user_用户ID` - 使用用户ID提及
- `@所有人` - @所有人（慎用）

### 方式3: 回复特定消息

```javascript
message({
  action: "send",
  target: "chat:CHAT_ID",
  message: "回复内容",
  replyTo: "MESSAGE_ID"  // 回复某条消息
})
```

## 🔧 交互模式

### 模式1: 主动发起对话

```
你 → @任务助手 请帮我创建一个新任务
```

### 模式2: 响应被@的消息

```
Kimi: @Kimi Claw 你能帮我整理一下文档吗？
你: 好的，我来帮你整理。
```

### 模式3: 多Agent协作

```
你 → @多维表格助手 请查询一下销售数据
你 → @阿里 请基于这些数据生成分析报告
```

## 📝 最佳实践

### 1. 明确提及对象

- ✅ 好: "@任务助手 请帮我创建一个优先级为高的任务"
- ❌ 差: "有人能帮我吗"

### 2. 简洁清晰的消息

- ✅ 好: "@Kimi 请总结一下上面的讨论"
- ❌ 差: "那个...就是...能不能..."

### 3. 适当的上下文

```
@多维表格助手 
基于刚才讨论的需求，请创建一个任务跟踪表，包含以下字段：
- 任务名称
- 负责人
- 截止日期
- 状态
```

### 4. 群聊礼仪

- 不要频繁@所有人
- 回复时保持礼貌
- 不相关的话题私聊
- 及时响应被@的消息

## 🔍 故障排查

### @不生效
- 检查用户名是否正确
- 确认对方在群聊中
- 尝试使用用户ID而非用户名

### 消息发送失败
- 检查群聊ID是否正确
- 确认有发送权限
- 查看错误日志

### 收不到回复
- 对方可能没有监听@消息
- 可能需要等待响应
- 尝试直接私聊
