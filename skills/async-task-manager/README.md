# Async Task Manager

通用异步任务管理框架，让所有耗时任务都能自动监控、闭环，并定期通知用户。

## 使用场景

- 长时间运行的数据处理任务
- 批量文件处理作业  
- 网络爬虫和数据抓取
- 机器学习模型训练
- 系统备份和维护任务
- 任何需要进度监控的后台任务

## 执行流程

1. **任务注册** - 通过命令行注册新任务，指定命令、名称和描述
2. **进程启动** - 自动使用 OpenClaw process 工具启动后台任务
3. **状态持久化** - 任务信息保存为 JSON 文件，包含状态、进度等
4. **自动监控** - 每30分钟自动检查任务状态（可配置）
5. **进度通知** - 通过 OpenClaw message 工具发送进度更新
6. **任务闭环** - 任务完成或失败时发送最终报告
7. **自动清理** - 过期任务自动清理（默认7天）

## 错误处理

- **任务注册失败**：检查任务名称是否重复，命令是否有效
- **进程启动失败**：确保 OpenClaw process 工具可用
- **通知发送失败**：检查 OpenClaw message 工具配置
- **权限问题**：确保技能目录有读写权限
- **Cron 配置失败**：手动运行 setup_cron.sh 脚本

## 配置选项

```json
{
  "default_notify_interval": 30,
  "task_storage_path": "./tasks",
  "notification_channel": "feishu",
  "max_concurrent_tasks": 10,
  "log_retention_days": 7
}
```

## 依赖工具

- `process` - OpenClaw 进程管理工具
- `message` - OpenClaw 消息通知工具  
- `cron` - 系统定时任务（用于自动监控）
- `bash` - 基础命令执行
- `read` - 文件读取
- `write` - 文件写入

## 安全保证

- **只读配置**：不修改 OpenClaw 核心配置
- **独立存储**：所有数据保存在技能目录内
- **权限控制**：只使用必要的 OpenClaw 内置工具
- **沙箱执行**：任务在隔离环境中运行
