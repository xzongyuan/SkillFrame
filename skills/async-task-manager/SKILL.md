# Async Task Manager

通用异步任务管理框架，让所有耗时任务都能自动监控、闭环，并定期通知用户。

## 使用场景

- 需要长时间运行的命令行任务（如数据处理、模型训练、文件转换等）
- 希望自动获得任务进度更新，而不需要手动检查
- 需要任务完成或失败时自动通知
- 希望任务状态能够持久化存储，即使重启也能恢复

## 执行流程

1. **任务注册**：使用 `task_manager.py register` 命令注册新任务
2. **进程启动**：系统使用 OpenClaw 的 `process` 工具启动后台任务
3. **状态监控**：每30分钟自动检查所有运行中的任务状态
4. **进度通知**：当需要时（基于配置的间隔），发送进度通知
5. **自动闭环**：任务完成后自动标记为完成状态
6. **清理过期**：自动清理7天前的旧任务记录

## 核心功能

- ✅ **任务注册**: 自动注册和追踪任意耗时任务
- ✅ **定期通知**: 每30分钟自动发送进度更新（可配置）
- ✅ **自动闭环**: 任务完成或失败时自动发送最终报告
- ✅ **持久化存储**: 所有任务状态持久化保存到 JSON 文件
- ✅ **通用接口**: 支持任何命令行任务
- ✅ **Cron 集成**: 自动设置后台监控任务
- ✅ **灵活配置**: 支持自定义通知间隔、保留天数等参数

## 错误处理

- **任务启动失败**：返回错误信息，不创建任务记录
- **进程异常终止**：在下次状态检查时标记为失败
- **通知发送失败**：记录日志但不影响主流程
- **文件读写错误**：使用安全的文件操作，避免数据损坏

## 配置选项

配置文件位于 `config.json`:
```json
{
  "default_notify_interval": 30,
  "task_storage_path": "./tasks",
  "notification_channel": "feishu",
  "max_concurrent_tasks": 10,
  "task_retention_days": 7
}
```

## 安全保证

- **只读配置**: 不修改 OpenClaw 核心配置
- **独立存储**: 所有数据保存在技能目录内
- **权限控制**: 只使用必要的 OpenClaw 内置工具
- **无外部依赖**: 不调用任何外部 API 或服务
